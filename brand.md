# Brand — RAG Document Chat

Chat with your own documents — hand-built retrieval-augmented generation.

_Generated by `brand-design` skill on 2026-07-19. Revised same day per follow-up request ("catchy, unique, premium" + light-only)._

## Palette — ChatGPT Light + Petrol Accent

**Vibe:** grayscale base, one deliberate accent, light-only
**Category:** ai / tech
**Direction:** Stark Minimal (per `design-taste`) — the narrowest palette that still functions, with exactly one accent reserved for primary actions

### History (why this looks the way it does)

1. Brand interview generated 6 "technical + premium" candidates (all blue-violet).
2. User asked for ChatGPT's actual look instead — pivoted to pure grayscale (`chroma 0` everywhere), both light and dark seeds derived.
3. User then asked for the design to feel "catchy, unique, premium" and to lock to light mode only. Pure grayscale alone read as a plain clone, not premium — per `design-taste`'s Stark Minimal direction, a single sparing accent is what separates "restrained" from "generic." Reintroduced exactly one accent color; removed the automatic dark-mode switch entirely.

### Seeds (light mode — the only mode now active)

| Role | OKLCH | Hex (approx.) |
|---|---|---|
| background | `oklch(0.995 0 0)` | `#fdfdfd` |
| card | `oklch(0.98 0 0)` | `#f8f8f8` |
| foreground | `oklch(0.16 0 0)` | `#0d0d0d` |
| **primary / accent** | `oklch(0.40 0.10 195)` | `#005758` (deep petrol/teal) |
| accent (soft tint) | `oklch(0.94 0.03 195)` | `#d5f2f1` |

Everything except `primary`/`accent` stays pure grayscale (`chroma 0`). The petrol hue (195°) was chosen deliberately to avoid the generic AI blue-violet (280°) — teal reads as distinctive and trustworthy without the "default AI gradient" smell.

**Dark-mode seeds still exist in `brand.md`'s git history** (commit before `a62eb0b`) but are no longer wired up — the app is light-only per explicit request. Re-run `brand-design` or restore that commit if dark mode is wanted later.

## Typography

**Geist** (sans) + **Geist Mono** (mono), unchanged from Phase 0 scaffold — close in spirit to ChatGPT's own typeface, so switching would have been low-value churn.

### Type scale in use

- Sidebar title: `text-sm font-semibold tracking-tight`
- Empty-state headline: `text-2xl font-semibold tracking-tight`
- Body / answers: `text-sm leading-relaxed`
- Meta / labels: `text-xs uppercase tracking-wide text-muted-foreground`

## Gradients

Not used. A gradient would fight the "one restrained accent, everything else neutral" intent.

## Motion

Single entrance animation (`animate-rise-in`: fade + 6px rise, 320ms, custom ease `cubic-bezier(0.16, 1, 0.3, 1)`) on new messages, answers, and citation lists. Wrapped in `@media (prefers-reduced-motion: no-preference)`. No other decorative motion — hover/active states use short opacity/scale transitions (150-200ms), never `transition-all`.

## Tone and voice

**Words to use:** direct, specific, factual — "answers are grounded in and cited from your files," not "unlock the power of your documents."

**Words to avoid:** hype words (revolutionary, supercharge, unleash), emojis in UI copy, exclamation marks, vague claims like "smart"/"powerful" without specifics.

**Voice example:** "Nothing relevant was found in your documents for this question." — states the honest outcome plainly, no apology padding.

## Usage dos and don'ts

**Do:**
- Use the CSS variables (`bg-background`, `text-foreground`, `bg-primary`, `bg-accent`, `text-accent-foreground`, `border-border`) everywhere — never hardcode a hex or a Tailwind color like `zinc-200`.
- Reserve `primary`/`accent` for genuinely primary moments: the send button, focus rings, citation markers, the streaming cursor, drag-active states. If more than a few elements on screen use it, it's being overused.
- Vary border-radius by role (pill for the input, `rounded-xl` for cards, `rounded-lg`/`rounded-md` for small chips/buttons) rather than one uniform value everywhere.
- Keep shadows low-opacity (4-8%) — elevation should feel quiet, not heavy.

**Don't:**
- Add a second accent color, or use the petrol accent decoratively (e.g. as a background wash) — it stays functional, not decorative.
- Reintroduce hardcoded `zinc-*`/`red-*`/`green-*` Tailwind classes for anything that should track the theme (status badges like "ready"/"failed" are an intentional exception — semantic state colors, not brand colors).
- Re-enable automatic dark mode without being asked — the app is light-only by explicit request.
- Override these tokens per-component — fix a wrong color once in `app/globals.css`.
